# QuoteGuard (Frontend) — AI 작업 가이드 (AGENTS.md)

> **이 문서는 프론트엔드(React + Vite) 전용 AI 컨텍스트 파일입니다.**
> 백엔드 서버(Spring Boot)는 별도 문서(`AGENTS.md` — Backend)에서 다룹니다.
> Codex / CodeRabbit 등이 이 문서를 읽고 일관된 방식으로 **프론트엔드 코드**를 작성·리뷰합니다.
> **모든 설명·주석·리뷰는 한국어로 작성합니다.**
>

---

## 1. 프로젝트 개요

**QuoteGuard**는 B2B 견적 관리 시스템이며, 본 문서는 그 **프론트엔드 클라이언트**를 다룹니다.

- **역할:** 백엔드 REST API(JSON)를 소비하는 SPA(Single Page Application)
- **빌드 도구:** Vite
- **성격:** 부트캠프 팀 프로젝트 — 과도한 설계보다 **실용적 스코핑** 우선
- **백엔드 연동:** Spring Boot REST API (별도 문서). 모든 데이터는 API를 통해 주고받음.

## 4. 코드 스타일 가이드

| 대상 | 규칙 | 예시 |
|------|------|------|
| 컴포넌트 / 컴포넌트 파일 | **PascalCase** | `QuoteList`, `QuoteList.jsx` |
| 함수 / 변수 | **camelCase** | `fetchQuotes`, `quoteId` |
| 커스텀 훅 | **`use` 접두사 + camelCase** | `useQuotes`, `useAuth` |
| 상수 | **UPPER_SNAKE_CASE** | `API_BASE_URL` |
| 일반 유틸 파일 | camelCase | `formatDate.js` |

기타 규칙:
- **함수형 컴포넌트 + Hooks** 만 사용. 클래스 컴포넌트 금지.
- 들여쓰기 2칸(스페이스). *(JS/React 관례 — 프로젝트 Prettier 설정 우선)*
- 한 컴포넌트는 한 가지 책임만. 너무 커지면 분리.
- JSX 안의 복잡한 로직은 컴포넌트 밖으로 추출(헬퍼 함수/훅).
- 매직 넘버·하드코딩된 문자열은 상수로 분리.
- 주석은 **의도(why)** 설명에 집중.

## 6. 백엔드 API 연동 (중요)

프론트엔드는 **자체 데이터를 갖지 않으며**, 모든 데이터는 백엔드 REST API를 통해 처리합니다.

- **Base URL** 은 환경변수로 관리: `.env` 의 `VITE_API_BASE_URL`. 코드에 하드코딩 금지.
- **API 호출은 `api/` 레이어로 일원화** — 컴포넌트에서 직접 `axios`/`fetch` 를 호출하지 말고, `api/` 의 함수를 통해 호출.
- **인증 연동** — 백엔드 Spring Security와 맞춰야 함.
  - ⚠️ **방식 확인 필요:** 쿠키(세션/JWT httpOnly) 기반인지, 토큰을 헤더로 보내는지 확인 후 통일.
  - 쿠키 기반이면 요청에 `withCredentials: true` (axios) 설정, 백엔드 CORS 허용 필요.
- **에러 처리** — 백엔드 전역 예외 응답(`@RestControllerAdvice`) 형식에 맞춰 공통 에러 핸들링.
- **로딩/에러 상태** — API 호출 시 로딩·에러·성공 상태를 UI에 반영.

## 7. 비즈니스 / UI 규칙 (중요)

백엔드 비즈니스 규칙을 UI에서도 동일하게 지킵니다.

1. **회원가입 페이지 없음** — 계정은 관리자가 생성. **셀프 가입 UI를 만들지 말 것.** (로그인 화면만 제공)
2. **역할(Role) 기반 UI 렌더링** — 권한 없는 메뉴·버튼·페이지는 숨기거나 접근 차단.
3. **`users.status` 반영** — 계정 상태(활성/대기/비활성 등)를 UI에 표시.
4. **완료된 견적서도 재작성 가능** — '완료' 상태에서도 수정 진입점을 막지 말 것.
5. **Feature 5 화면 **
   - 견적서 **PDF 미리보기 / 다운로드**
   - **이메일 발송** 화면 + 발송 결과 피드백
   - **발송 이력** 조회 화면
   - **엑셀 다운로드** 버튼
   - 한글 데이터(거래처명·품목 등)가 PDF·엑셀에서 깨지지 않는지 확인.

## 8. 코드 품질 규칙

- **API 호출은 `api/` 레이어로 분리** — 컴포넌트는 화면에 집중.
- **공통 컴포넌트 추출** — 버튼·모달·테이블 등 반복 UI는 재사용 컴포넌트로.
- **불필요한 리렌더 주의** — 적절한 상태 분리, 필요 시 `useMemo`/`useCallback` (남용은 금지).
- **과도한 설계 지양** — 부트캠프 스코프에 맞게 간결하게. 불필요한 전역 상태·추상화 금지.
- **`key` 명시** — 리스트 렌더 시 안정적인 `key` 사용 (index 남용 지양).
